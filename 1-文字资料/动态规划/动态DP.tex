\textbf{动态 DP}

动态 DP 处理的是“树上 DP + 单点修改”。其本质不是某个具体 DP，而是把局部转移写成\textbf{可结合的广义矩阵乘法}，再用重链剖分把“根到叶的递推”改写成“链上矩阵连乘”。

\textbf{状态抽象}

设 $F_u\in V^k$ 为点 $u$ 的 DP 状态。若能把除重儿子外的所有贡献都吸收到局部矩阵 $G_u$ 中，使
\[
F_u = G_u \otimes F_{son(u)},
\]
其中 $\otimes$ 满足结合律，则可做动态 DP。若 $u$ 无重儿子，则把 $F_{son(u)}$ 视为固定边界向量 $E$。

\textbf{重链表示}

设一条重链为
\[
u_1 \to u_2 \to \cdots \to u_m,
\]
其中 $u_{i+1}=son(u_i)$。则由结合律，
\[
F_{u_1}
=
G_{u_1}\otimes G_{u_2}\otimes \cdots \otimes G_{u_m}\otimes E.
\]

因此整条重链只需维护链上矩阵乘积
\[
P_{u_1}
=
G_{u_1}\otimes G_{u_2}\otimes \cdots \otimes G_{u_m}.
\]
一旦得到 $P_{u_1}$，链顶状态便可写成
\[
F_{u_1}=P_{u_1}\otimes E.
\]

\textbf{使用线段树维护 $G$}

在线段树中维护重链上各点的 $G_u$，每个线段树节点维护对应区间的矩阵乘积。若按 DFS 序从浅到深存放，则区间合并顺序必须是
\[
P_{[l,r]} = P_{[l,mid]}\otimes P_{[mid+1,r]},
\]
顺序不能反，因为矩阵乘法通常不满足交换律。于是单点修改某个 $G_u$ 后，可以在 $O(\log n)$ 时间内求出整条重链的新乘积。

\textbf{修改传播}

设修改发生在点 $x$。它会直接改变某些局部参数，从而改变 $G_x$。更新 $G_x$ 后，$x$ 所在重链的总乘积 $P_{top(x)}$ 也随之改变，于是链顶状态
\[
F_{top(x)}
=
P_{top(x)}\otimes E
\]
发生变化。

此时有两个关键点：

\begin{itemize}
	\item 修改传播时，当前重链上除了被改到的那个点外，其余点的 $G$ 都不会改变，所以不需要逐点修改，只需在线段树里单点改一个 $G$，再重新取整条链的乘积。
	\item 链顶 $top(x)$ 对其父亲 $fa(top(x))$ 来说是一个轻儿子，因此真正需要继续改写的是父点的 $G_{fa(top(x))}$，然后再跳到上一条重链继续处理。
\end{itemize}

因此一次修改的过程是：

\begin{itemize}
	\item 单点修改当前点的 $G$；
	\item 线段树查询当前重链乘积，得到链顶状态的新值；
	\item 用链顶状态的改变量去更新父点的 $G$；
	\item 沿轻边向上跳，直到根。
\end{itemize}

由于根到任一点经过的轻边数为 $O(\log n)$，每次链内单点修改或区间乘积查询又是 $O(\log n)$，故总复杂度为
\[
O(\log^2 n).
\]

\textbf{速记}

\begin{itemize}
	\item 先把树上 DP 写成 $F_u = G_u \otimes F_{son(u)}$。
	\item 用线段树维护重链上的 $G$ 的连乘。
	\item 修改时只改单点的 $G$；同链其他点的 $G$ 不变。
	\item 真正向上传播的是链顶状态对父点轻儿子贡献的变化。
\end{itemize}
